riannah's garden

Product goal

Build a secure, simple household plant management app for closed beta users. Users can track their plants, watering schedule/history, and repotting schedule/history. Architecture should stay minimal, dependency-light, and Cloudflare-native.

v1 scope (committed)

  • Closed-beta authentication (no public sign-up)
  • Role-based access: admin and user
  • Admin can create users (admin/user roles)
  • User can add/edit/archive their plants
  • User can set watering schedule and log watering events
  • User can see what needs watering now/upcoming
  • User can set repotting schedule and log repotting events
  • Plant profile captures enough metadata for future care guidance enrichment
  • Separate dev deployment for safe testing before production

Out of scope (v1)

  • Public registration
  • Advanced automated plant care recommendations
  • Push/email reminders
  • Multi-tenant organizations/teams

Proposed architecture (simple + secure)

  • Frontend: React + Vite (minimal dependency policy)
  • Hosting: Cloudflare Pages
  • Backend API: Cloudflare Workers
  • Data: Cloudflare D1 (SQL, simple relational model)
  • Auth: HttpOnly session cookies issued by Worker
  • Environments: separate dev and prod deployments + isolated data

Initial data model

  • users: id, email, password_hash, role, status, created_at
  • plants: id, owner_user_id, nickname, species_common, species_scientific, acquired_on, notes, next_water_on, next_repot_on, created_at
  • plant_events: id, plant_id, owner_user_id, type (water|repot), occurred_on, next_due_on, created_at

Species fields are included now so we can attach care guidance datasets later without schema churn.

Security baseline

  • Argon2id (or bcrypt if platform constraints) for password hashing
  • HttpOnly + Secure + SameSite session cookies
  • Server-side role enforcement on every admin route
  • Row ownership checks: users can only access their own plants/events
  • Input validation at API boundary
  • Audit trail for admin user creation actions
  • No secrets in repo; all secrets via Cloudflare/GitHub env secrets

Delivery phases

Phase 0: foundation

  • Create riannahs-plants repo and baseline CI
  • Set up Cloudflare dev + prod deployments and env vars
  • Create ADR directory and seed architecture decisions

Phase 1: auth + RBAC

  • User login/logout
  • Admin-only user creation flow
  • Role-protected API routes

Phase 2: plants + schedules

  • CRUD plants
  • Set watering/repotting schedules
  • Dashboard view: due and upcoming tasks

Phase 3: event tracking + polish

  • Log watering and repotting events
  • History views per plant
  • Basic UX polish + v1 hardening

UI theme direction

  • Nature-inspired visual language
  • Primary palette: olive / greens
  • Supporting neutrals: tan, brown, muted earthy red
  • Tone: calm, grounded, minimal

Dev deployment strategy (early)

  • Production: deploy from main
  • Development: deploy from develop branch
  • Separate D1 database for dev vs prod
  • Separate session secrets per environment
  • PRs validated against dev before merge to main
